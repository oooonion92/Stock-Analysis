# 每日复盘流程说明

更新日期：2026-06-18

本文档用于说明 `D:\Projects\Stock-Analysis\02_daily_replay` 当前已经跑通的每日复盘流程。目标不是“写一篇盘后感想”，而是把每天复盘固定成一条可重复执行的流水线：

1. 读取本地行情与全 A 数据
2. 用缠论脚本完成量化结构拆解
3. 对指数、题材、高手发言、跟踪标的做 AI 精炼判断
4. 输出 `md / json / html`
5. 同步云端阅读版 HTML

## 1. 当前流程原则

现在的复盘框架分成两层：

1. 量化底稿层
   由 `chanlun_replay_plan.py` 读取 5m/30m 数据、全 A 数据、watchlist 配置，生成结构化底稿。

2. AI judgment 层
   不再让最终看板只靠固定句库或阈值模板输出，而是在量化底稿之上，优先读取当天的 AI judgment 文件，输出更贴合当日结构的结论。

这点是 2026-06-17 之后最重要的更新：

- 指数的“明日路径”不再只靠硬编码阈值判断。
- 个股卡片也不再只靠规则句库生成，而是优先读取 AI 精炼后的 `takeaway / summary / structure / theme_plan`。
- 看板的最终目标是“先量化拆解，再 AI 深思后落地”，不是单纯量化吐话。

## 2. 数据入口

### 2.1 高频行情数据

目录：

```text
D:\OneDrive\Stock\details
```

核心文件包括：

- `sh000001_5Min_MaxAvailable.csv`
- `sh000001_30Min_MaxAvailable.csv`
- 跟踪标的对应的 `5Min / 30Min` CSV

用途：

- 判断指数与个股的 5m/30m 中枢位置
- 判断是否处于离开段、中枢内、跌回中枢等结构
- 计算 VWAP、VWAP 偏离
- 计算 DIF、MACD、斜率、近端分型等量化信息

### 2.2 全 A 市场数据

文件：

```text
D:\OneDrive\Stock\details\全部Ａ股YYYYMMDD.xlsx
```

用途：

- 全 A 上涨 / 下跌家数
- 上涨占比、中位涨幅
- 主力净额、成交额
- 强势行业、高成交行业、主题映射
- 跟踪标的的题材归因

注意：

- 必须使用当天文件。
- 如果当天 `全部Ａ股YYYYMMDD.xlsx` 缺失，题材生态模块必须明确提示缺数据，不能拿旧文件冒充当天。

### 2.3 高手发言素材

目录：

```text
D:\OneDrive\Stock\Replies collect
```

当前复盘主要使用：

- `今日汇总.md`

可辅助参考但不作为前置依赖：

- `短线高手汇总.md`
- `趋势高手汇总.md`
- `最近7天汇总.md`

用途：

- 提取当天市场共识、分歧、风险、执行纪律
- 精选有交易意义的原话摘录
- 生成当日 `expert_digest`

## 3. 配置入口

主配置文件：

```text
D:\Projects\Stock-Analysis\02_daily_replay\plans\watchlist_config.json
```

当前应维护的内容：

- `index`：默认指数
- `symbols`：当天真正需要复盘的跟踪对象
- `market_context`：当天额外补充的盘面上下文
- 账户或标的信息（如果需要）

维护规则：

- 已清仓且不再跟踪的标的，从 `symbols` 移除
- 新增趋势标的或 ETF，先加入 `symbols`
- 没有对应 5m/30m 数据的标的，不应进入当天复盘

## 4. 核心脚本与运行方式

主脚本：

```text
D:\Projects\Stock-Analysis\02_daily_replay\tools\chanlun_replay_plan.py
```

脚本当前会做这些事：

5. 读取当天 `expert_digest`
6. 读取当天 `index_judgment`
7. 读取当天 `stock_judgments`
8. 输出 `md / json / html`
9. 同步写入云端阅读版 HTML

## 5. 新增的 AI judgment 文件

这是昨天流程更新的核心。

### 5.1 高手发言精炼

文件：

```text
plans\YYYYMMDD_expert_digest.json
```

作用：

- 对 `今日汇总.md` 做 AI 精炼
- 生成盘面结论、交易类型、主线焦点、执行结论
- 输出核心判断、明日过滤器、机会线索、风险线索、精选摘录

说明：

- 看板优先读取这个文件
- 如果没有这个文件，会退回到原始汇总或配置中的较弱版本

### 5.2 指数 judgment

文件：

```text
plans\YYYYMMDD_index_judgment.json
```

作用：

- 在量化底稿基础上，对指数做 AI 二次判断
- 重点覆盖：
  - `summary`
  - `structure`
  - `paths.red / yellow / blue`

这一步用于解决旧流程里“明日路径过于机械、总是被阈值写死”的问题。

现在看板会优先读取这里的内容，而不是直接套固定路径模板。

### 5.3 个股 judgments

文件：

```text
plans\YYYYMMDD_stock_judgments.json
```

作用：

- 对每个跟踪标的做 AI 精炼输出
- 当前字段建议统一为：

```json
{
  "date": "2026-06-17",
  "updated_at": "2026-06-17 19:15",
  "judgments": {
    "sz002463": {
      "takeaway": "...",
      "summary": "...",
      "structure": "...",
      "theme_plan": "..."
    }
  }
}
```

看板当前会优先读取这些字段：

- `takeaway`：卡片顶部一句话
- `summary`：结论
- `structure`：结构
- `theme_plan`：题材与执行

这一步用于解决旧流程里“个股卡片只是句库复读”的问题。

## 8. 每日执行清单

### 8.1 复盘前检查

```text
1. details 目录是否已有当天指数 5m/30m 数据
2. details 目录是否已有当天跟踪标的 5m/30m 数据
3. 是否已有当天 全部Ａ股YYYYMMDD.xlsx
4. Replies collect\今日汇总.md 是否已更新
5. watchlist_config.json 中 symbols 是否是当天真实跟踪对象
6. 是否需要新增当天的 expert_digest / index_judgment / stock_judgments
```

### 8.2 复盘生成后检查

```text
1. 输出日期是否正确
2. 全 A 数据是否来自当天文件
3. 指数结论是否围绕当天结构，而不是旧中枢位置
4. 明日路径是否体现“先量化、后 AI judgment”
5. 个股卡片是否已显示 AI 精炼内容，而不是纯规则句
6. 清仓标的是否已移除
7. 新增标的是否已进入看板
8. 云端 HTML 是否已更新到 D:\OneDrive\Stock\Daily review
```

## 9. 建议的日常工作顺序

每天建议按这个顺序走：

1. 更新 `watchlist_config.json`
2. 确认 `details` 和 `今日汇总.md` 已更新
3. 先跑量化复盘脚本，拿到底稿
4. 结合底稿与当天素材，补写：
   - `YYYYMMDD_expert_digest.json`
   - `YYYYMMDD_index_judgment.json`
   - `YYYYMMDD_stock_judgments.json`
5. 再次运行脚本，生成最终版 html
6. 以云端 HTML 作为当天主要阅读版

## 10. 现在这份说明的结论

当前项目里的“每日复盘”已经不是单一路径：

- 不是纯缠论量化输出
- 也不是纯主观复盘

而是：

```text
本地数据 -> 缠论量化拆解 -> AI judgment 精炼 -> md/json/html -> 云端 HTML 阅读
```

这就是 2026-06-17 之后应长期使用的新流程。
