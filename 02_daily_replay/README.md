# Daily Replay

这里放每日复盘、次日预案、HTML 看板和方法论资料。

## 精炼方法论

- `plans/stock_trading_methodology.md`: 高手笔记与缠论要点提炼后的操盘方法论。
- `plans/replay_method_prompt.md`: 每次复盘时应优先调用的执行准则。

核心口径：

- 先分交易类型：短线 / 趋势 / 打野 / 防守。
- 再过三道过滤器：结构、情绪/主线、仓位。
- 缠论只确认结构，不单独构成买入理由。
- 新增买入必须能回答级别、市场理由、结构买点、失效卖点和仓位原因。

## 原始素材

`source_notes/` 保留原始高手笔记和缠论资料：

- 拨佬操作集锦
- 狼大投资心法
- 股票论坛发言操作逻辑分析
- 缠论 108 课

日常复盘优先用精炼方法论；需要追溯依据或继续提炼时再读原始素材。

## 每日看板操作指南

2026-06-11 起，每日 HTML 看板采用三段阅读顺序：

1. `盘面与题材复盘`：把大盘结构、全市场题材生态、跟踪对象发言精炼合并到同一个模块里阅读。不要把高手发言摘要单独堆成另一个盘面判断层，避免和大盘/题材重复。
2. `有用摘录`：放在盘面与题材复盘之后、标的复盘之前，作为判断证据墙。摘录必须保留原话、背景、交易含义、明日动作和原帖链接；只保留能转化为交易含义和明日动作的内容，闲聊、纯情绪、无法对应主线/风险/执行的内容不进入看板。
3. `标的复盘`：只保留复盘对象，不追踪仓位、成本、浮盈。个股卡片保持短句结构：结论、结构、题材与执行，每段两三句话即可。

数据融合规则：

- 高频结构来自 `D:\OneDrive\Stock\details` 下的指数和个股 5m/30m CSV。
- 全市场题材生态优先读取 `D:\OneDrive\Stock\details\全部Ａ股YYYYMMDD.xlsx`；如果当天文件缺失，题材生态必须明确提示缺数据，不用旧文件冒充当天结论。
- 高手发言摘要不要求用户另行精炼。复盘时先读取 `D:\OneDrive\Stock\Replies collect\今日汇总.md`，由当前复盘对话先做 AI 精炼，再落成 `plans/YYYYMMDD_expert_digest.json` 供当天看板优先读取；若已有 `今日总结看板_YYYYMMDD.html`，只作为辅助参考，不作为前置依赖。
- 每次生成会同时写项目内归档 HTML 和云端阅读版 `D:\OneDrive\Stock\Daily review\YYYYMMDD_每日复盘.html`。
- 新增买入继续执行三道过滤器：结构、情绪/主线、仓位。缠论只确认结构，不单独构成买入理由。
